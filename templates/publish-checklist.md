# Checklist Para Publicar Un Nuevo Visual

Usa este checklist antes de agregar o promocionar un visual en el repositorio.

## Archivos

- [ ] Carpeta del visual creada con nombre claro.
- [ ] Archivo `.pbix` incluido.
- [ ] Archivo `.json` incluido.
- [ ] Dataset de ejemplo `.xlsx` incluido.
- [ ] Thumbnail `.png` incluido.
- [ ] README especifico del visual incluido.

## Calidad Del Dataset

- [ ] Fechas guardadas como tipo fecha.
- [ ] Acentos y caracteres revisados.
- [ ] IDs unicos.
- [ ] Columnas requeridas documentadas.
- [ ] Valores categoricos estandarizados.
- [ ] Datos de ejemplo suficientes para probar el visual.

## Calidad Del Visual

- [ ] JSON probado en Deneb.
- [ ] Version de Vega o Vega-Lite documentada.
- [ ] Tooltips revisados.
- [ ] Colores legibles.
- [ ] Estados y prioridades visibles.
- [ ] Thumbnail actualizado desde el visual real.

## Documentacion

- [ ] Caso de uso explicado.
- [ ] Audiencia definida.
- [ ] Instrucciones de instalacion claras.
- [ ] Tabla de columnas incluida.
- [ ] Limitaciones o supuestos documentados.
- [ ] Creditos incluidos.

## Publicacion

- [ ] README principal actualizado.
- [ ] `VISUAL_CATALOG.md` actualizado.
- [ ] Version o release notes preparados.
- [ ] Cambios revisados con `git status` y `git diff`.
